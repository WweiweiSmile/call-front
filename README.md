# Call 游戏存取分管理系统

基于 Taro + React + NutUI 的 H5 移动端游戏存取分管理系统。

## 业务概述

存取分业务是游戏场景中的核心业务。用户可以创建游戏，其他用户加入游戏后，在游戏过程中进行积分的存入和取出操作。每场游戏的积分完全独立管理，只为当场游戏负责，场次间积分不能转移。

**重要规则：**
- 积分只为当场游戏负责，场次间积分不能转移
- 存分和取分除了用户自己能操作外，游戏创建人也能代理操作
- 只需要当场次平衡校验，不需要全局平衡校验

## 技术栈

- **框架**: Taro 4.1.11
- **UI 库**: NutUI React Taro 2.6.14
- **语言**: TypeScript
- **样式**: Less

## 页面功能

### 1. 游戏列表页 (`pages/games`)
- 显示进行中的游戏
- 显示即将开始的游戏
- 搜索游戏
- 创建游戏
- 加入游戏

### 2. 我的场次页 (`pages/my-games`)
- 显示进行中的场次
- 显示我创建的游戏
- 显示历史场次
- 显示场次余额和平衡状态

### 3. 场次详情页 (`pages/game-detail`)
- 显示场次积分状态
- 存分操作
- 取分操作
- 查看交易记录
- 游戏创建者可以管理所有参与者
- 支持代理操作

### 4. 个人中心页 (`pages/profile`)
- 用户信息展示
- 统计数据展示
- 菜单导航

## 业务规则说明

### 积分操作权限

- **用户自己**：可以操作自己的积分
- **游戏创建者**：可以代理操作该游戏中任何参与者的积分

### 平衡校验

- **场次平衡**：场次存分总量 - 场次取分总量 = 0 为平衡
- **校验范围**：只需要当场次平衡校验，不需要全局平衡校验

### 场次独立性

- 游戏场次间积分不能转移
- 积分只为当场游戏负责
